## How This Document Works

Every content piece moves through these stages:

```
Identified → Researched → Outlined → Drafted → Reviewed → Published → Monitoring
```

**Identified** = we know we want to write it, but haven't done keyword research yet
**Researched** = keyword research done, competition assessed, angle decided
**Outlined** = structure and key points mapped, internal linking plan set
**Drafted** = written and math-verified, sitting in content/ directory
**Reviewed** = Justin has done voice/accuracy pass
**Published** = live on stickapp.golf, submitted to Search Console
**Monitoring** = tracking rankings and performance in SEO Scoreboard

The goal is to always have 3-4 pieces in "Researched + Outlined" status so we can draft on demand without starting from scratch.

## Pounce List — Pre-Loaded Triggers

These are content frameworks ready to activate when an external event creates a search spike. We can't predict *what* will trend, but we can predict the *types* of things that trend in golf and have outlines ready.

| Trigger Type | Example | Pre-Loaded Response | Lead Time |
|-------------|---------|-------------------|-----------|
| **Major tournament week** | Masters, US Open, Ryder Cup, PGA Championship | "Best Games to Play During [Tournament]" — template matches Masters Week post format | 2 weeks before |
| **Celebrity golf event goes viral** | The Match, Capital One's The Match, celebrity scramble | "How [Format] Actually Works: Rules & Settlement" — explain the format being played | 48 hours |
| **USGA rule change affecting scoring** | Handicap calculation update, stroke allocation change | "How [Rule Change] Affects Your Golf Bets" — explain impact on each game type | 1 week after announcement |

| **Viral golf moment** | Hole-in-one bet pays out, famous golf wager story | "The Math Behind [Moment]: How Golf Bets Actually Work" — educational angle | 48 hours |

**How the Pounce List works:** When a trigger fires, we don't write from scratch. We grab the template, customize for the specific event, and publish fast. The weekly briefing scan checks for active triggers.

## Queue Management Rules

1. **Always maintain 3-4 pieces at "Researched + Outlined" stage.** This means any session can produce a draft without cold-starting research.
2. **Tier 1 before Tier 2 before Tier 3.** Exceptions allowed for seasonal timing (see Content Calendar).
3. **Guide expansions (Tier 4) can happen alongside any other work.** They're quick and compound existing content value.
4. **When Search Console data arrives, re-prioritize based on what's actually working.** A page ranking #12 for a valuable keyword is more urgent than writing something new.
5. **Every new piece must link to at least 2 existing pages, and at least 2 existing pages should link back to it.** Plan the internal linking before drafting.
6. **After publishing, revisit this document and update status.** Don't let it go stale like the original content calendar did.
